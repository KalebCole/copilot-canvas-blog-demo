# copilot-canvas-blog-demo

demo repository for the GitHub Copilot Canvas blog article, very simple, not trying to say much

## Repository Issues Kanban

1. Open this repository in a GitHub Copilot desktop project session.
2. Open **Repository Issues Kanban** from the canvas picker, then select **Refresh** to load open GitHub issues.
3. Drag cards between **Backlog**, **Plan**, **Ready**, **Implement**, and **Done**. Moving a card into **Plan** starts Copilot's background planning workflow; the other moves are manual.

The board's lane placement, filters, and agent activity are stored locally by the extension. It reads the repository's open GitHub issues but does not change their GitHub status.
